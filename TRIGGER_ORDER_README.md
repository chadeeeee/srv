# Trigger Order - Quick Start

## 🎯 Що реалізовано

Бот тепер використовує **TRIGGER ORDER** (Conditional Order) **ТІЛЬКИ ДЛЯ SHORT** позицій у стратегії Gravity2.

## 📋 Коротко про логіку

### Gravity2 SHORT (після перекупленості) - TRIGGER ORDER ✅
1. Отримали сигнал "LONG (підтримка)" з каналу
2. Чекаємо RSI >= 70 (перекупленість)
3. Коли RSI досяг 70+:
   - Беремо останню закриту свічку
   - **Trigger ставимо на LOW свічки**
   - Stop Loss на екстремумі свічки
4. **Bybit автоматично виконає SHORT коли ціна впаде до LOW свічки**

### Gravity2 LONG (після перепроданості) - LIMIT ORDER 📊
1. Отримали сигнал "SHORT (опір)" з каналу
2. Чекаємо RSI <= 30 (перепроданість)
3. Коли RSI досяг 30-:
   - Беремо останню закриту свічку
   - **Одразу ставимо LIMIT ORDER на поточній ціні**
   - Stop Loss на екстремумі свічки
4. **Виконується як звичайний LONG вхід**

## ⚡ Чому тільки SHORT через trigger?

- **SHORT через trigger**: чекаємо підтвердження падіння ціни до LOW свічки
- **LONG без trigger**: входимо одразу, бо перепроданість вже відбулась
